# 📘 2. FRD (Functional Requirements)

## 🔐 AUTH SYSTEM
*   **Signup/Login:** Email + Password (Optional: OTP/Social login).
*   **Role-Based Access Control (RBAC):**
    *   **Student:** Browse, download, comment, and rate.
    *   **Contributor:** Upload notes, manage own content.
    *   **Admin:** Full dashboard access, moderation, user management.

## 📚 NOTES MODULE
*   **Upload:** Support for PDF and images.
*   **Filtering System:**
    *   College → Course → Semester → Subject.
*   **Interaction:** Ratings (1-5 stars) + text comments.

## 👥 COMMUNITY MODULE
*   **Posts:** Users can post doubts or start discussions.
*   **Voting:** Upvote/Downvote system for quality posts.
*   **Threading:** Nested comments for deep discussions.

## 💬 CHAT MODULE
*   **1:1 Chat:** Direct messaging between students/contributors.
*   **Phase 2:** Group chats for subjects or colleges.
*   **File Sharing:** Share notes directly in chat.

## 👨‍🏫 PROFESSOR MODULE
*   **Database:** List of professors by college/department.
*   **Rating Parameters:** Teaching quality, Strictness, Marks, Attendance.
*   **Reviews:** Text-based detailed student feedback.

## 🏫 COLLEGE MODULE
*   **Catalog:** Directory of colleges.
*   **Reviews:** Institutional level feedback.
*   **Aggregator:** All notes linked to specific colleges.

## 👤 PROFILE MODULE
*   **Identity:** Basic info, college affiliation.
*   **Analytics:** Record of uploaded notes and community activity.

## 🛠️ ADMIN PANEL
*   **Approval Queue:** Review and approve/reject notes.
*   **Moderation:** Delete offensive posts/comments.
*   **Management:** User banning/role updates.
